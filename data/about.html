01 — Overview

INTRODUCTION

The advancement of the Internet of Things (IoT) and embedded systems has opened new possibilities in human–machine interaction, robotics, and remote control technologies. This project aims to design and implement a robotic hand that can be wirelessly controlled through a web-based interface.

The project utilises one ESP32 microcontroller embedded inside the robotic arm. Beyond physical control, the system also features a web server hosted directly on the ESP32 — allowing any standard web browser on the local network to command the hand in real time.

The IoT Robotic Hand serves as a prototype for future applications in assistive technology, teleoperation, prosthetics, and remote robotic systems. It showcases how affordable hardware and IoT connectivity can be combined to create interactive, practical solutions in modern robotics.

02 — Build Progress

HAND DESIGNS

Every build has a story. Here is the full journey from the first wiring session to a fully assembled, ESP32-powered robotic arm.

P.01
Phase 01 — Electronics Prep

Wiring the Brain

Before any plastic was printed, the hardware foundation had to be laid. The ESP32 microcontroller was fitted with female Dupont headers and wired up with colour-coded jumper cables — one channel per servo. The rainbow of wires maps directly to the five fingers: each colour representing a dedicated PWM signal line. Getting the pinout right at this stage was critical. A mislabelled wire here would mean a misfiring finger later.

ESP32 Dev Board — GPIO headers & PWM signal lines
P.02
Phase 02 — First Assembly

Hand Model v1 — Palm & Thumb Servo

The first 3D-printed hand arrived in skin-tone PLA. At this stage the focus was purely mechanical: verifying that the jointed finger segments articulate correctly and that the palm provides enough real-estate to house a servo. A single MG90S micro-servo was mounted on the palm to test thumb actuation. With four fingers naturally extended and the thumb independently controlled, the proof-of-concept was already taking shape.

P.03
Phase 03 — Design Iteration

Adding the Forearm — Internal Architecture

The hand alone was not enough. A full forearm section was designed and printed to act as the electronics enclosure, keeping the ESP32 and wiring hidden inside the body. This photo captures the design-review moment: the forearm shell opened up, the wrist joint and rotation disc components laid out separately, and the ESP32 board placed above to plan fitment. The hollow interior was engineered with mounting tabs to keep the board secure without adhesive.

v2 · Forearm
Design review — forearm shell, wrist rotation discs, and ESP32 fitment planning
P.04
Phase 04 — Structural Assembly

Wrist Joint & Forearm Integration

With the shell design confirmed, the arm was assembled for the first time as a single structural unit. The side view reveals the wrist connector bridging hand and forearm, the internal mounting rails clearly visible through the open back panel, and the servo bracket locked into position at the palm. This angle was used to check alignment tolerances — any twist or offset in the wrist joint would accumulate into positioning error at the fingertips.

v3 · Whole Hand
Side profile — wrist joint, internal rail system, and palm servo bracket
P.05
Phase 05 — Electronics Integration & Final Build

ESP32 Mounted — The Complete System

The final stage: the ESP32 was slid into its forearm cradle and the arm closed up. The left photo shows the board settled at the base of the forearm cavity, USB port accessible from the bottom for flashing and power. The right photo captures the finished arm standing upright — all five fingers extended, the wrist servo visible through the palm opening, and the ESP32 peeking out from below. At this point the web server was live, the hand was online, and control via any browser on the local network was fully operational.

03 — Software

PROGRAMMING

Backend

C++ on ESP32

Implements an onboard web server using C++ to handle HTTP requests, control servo motors, and manage communication between the browser and the robotic hand.

  • › Wi-Fi connection & HTTP callbacks
  • › GPIO & PWM servo control
  • › Serial command logging
Frontend

HTML · CSS · JavaScript

Provides a responsive web interface for controlling the robotic hand from any browser. JavaScript sends commands to the ESP32's server to move servos and display feedback dynamically.

  • › Real-time sliders & gesture buttons
  • › HTTP requests to ESP32 server
  • › Live status & feedback display
